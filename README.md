# 牛市热度指数 (Bull Market Heat Index)

![Python](https://img.shields.io/badge/Python-3.10-blue)
![GitHub Actions](https://img.shields.io/badge/GitHub%20Actions-Automated-brightgreen)
![License](https://img.shields.io/badge/License-MIT-lightgrey)

**量化A股市场冷热程度，辅助判断顶部与底部。**

**取值范围**：0 – 100  
**更新频率**：每个A股交易日北京时间 20:00  
**数据源**：akshare（免费，零依赖）

## 热度等级

| 指数区间 | 等级 | 解释 |
|---------|------|------|
| 0–20 | 很冷 🥶 | 历史极值低位，长期布局窗口 |
| 20–40 | 偏冷 ❄️ | 市场偏冷，关注低估机会 |
| 40–60 | 适中 ✅ | 无显著超买超卖，常规仓位 |
| 60–80 | 偏热 🔥 | 短期过热风险上升，注意控仓 |
| 80–100 | 很热 🚨 | 逃顶风险显著，建议降低仓位 |

## 指数计算方法

**公式**：`当日指数 = Σ (每个指标的当日历史百分位 × 权重)`

### 权重规则

采用 **双梯队加权**，核心指标更高敏感度：

| 梯队 | 指标 | 权重 |
|------|------|:----:|
| **核心梯队**（快速响应+长历史） | 沪深换手率、两融余额市值比、大盘估值-PE、股债利差、成交额M2比、巴菲特指标 | **各 12%** |
| **辅助梯队**（短历史/慢响应） | 上交所个人开户数、存款市值比、偏股基金三年年化、股债基金比 | **各 7%** |

- **缺失值处理**：缺失指标的权重按比例均分给当日有效指标，保持总权重归一化
  - 例如核心指标 A（12%）缺失 → 剩余 9 个指标的总权重 = 88%，每个有效指标的实际权重 = 原始权重 / 88%

### 百分位定义

`历史百分位 = (小于当前值的历史数据个数) / (历史数据总个数) × 100`

- **严格小于**：等于当前值时不计数，因此百分位取值范围 0–100
- 百分位 = 100 仅在当前值大于全部历史值时出现
- **历史窗口**：从该指标有数据的第一天起，包含全部历史数据
- 百分位一经计算不再追溯重算（除非修正原始数据）

### 设计原则

指数的核心思路是 **"将多个不同维度的市场指标统一到百分位尺度上，再用差异化权重合成"**。快速响应且历史长久的指标享有更高权重（各12%），短历史或慢响应指标权重较低（各7%）。当多个核心指标同时处于历史极端位置时，指数给出强烈信号。

## 指标清单 — 各指标含义与逻辑

| # | 指标 | 权重 | 梯队 | 频率 | 核心逻辑 | 牛市顶特征 | 熊市底特征 |
|---|------|:----:|:----:|------|---------|-----------|-----------|
| 1 | **巴菲特指标**<br>(A股总市值/滚动GDP) | **12%** | 核心 | 月→日 | **衡量股市总市值与国民经济总产出的比值**。巴菲特认为该指标 >100% 时市场过热，<50% 时被低估。A股历史在 40%~120% 之间波动。 | 总市值膨胀远超GDP增速，比值攀升至历史高位 | 市值缩水，比值回落至低位 |
| 2 | **成交额M2比**<br>(两市成交额/M2余额) | **12%** | 核心 | 月→日 | **成交额与货币总量的比率**。剔除了货币宽松带来的成交额自然增长，衡量的是"资金参与股市的意愿度"。M2衡量的全市场资金量，成交额是流入股市的部分。 | 成交额增速远超M2增速，资金集中涌入股市 | 成交额萎缩，资金撤离 |
| 3 | **沪深换手率**<br>(成交额/流通市值) | **12%** | 核心 | 日 | **单位流通市值的交易活跃度**。换手率越高，说明短期交易越频繁，情绪越亢奋。A股历史换手率在 0.3%~8% 之间，极值时可达 10%+。 | 换手率飙升，短线投机盛行，筹码频繁交换 | 换手率低迷，市场交投冷清 |
| 4 | **两融余额市值比**<br>(融资融券余额/总市值) | **12%** | 核心 | 日 | **杠杆资金参与度**。融资融券是市场上最敏感的杠杆资金，余额占比越高说明市场风险偏好越强。2015年牛市该指标冲到历史极值。 | 杠杆资金疯狂涌入，两融余额占市值比飙升 | 去杠杆过程，融资余额大幅萎缩 |
| 5 | **大盘估值-PE**<br>(上证指数平均市盈率) | **12%** | 核心 | 月→日 | **核心宽基指数的估值水平**。使用上证指数月频平均市盈率（1997~至今），历史在 10x~60x 之间波动。PE越高，市场越贵，未来预期回报越低。 | PE扩张至历史高位，泡沫化 | PE压缩至历史低位，价值洼地 |
| 6 | **股债利差**<br>(1/PE − 10年国债收益率) | **12%** | 核心 | 日 | **股票相对债券的性价比**。也叫"股权风险溢价"(ERP)。利差越大，股票相对债券越有吸引力；利差越小甚至为负，说明股票性价比极低。 | 利差收窄甚至为负 → 股票太贵不如买债券 | 利差走阔 → 股票性价比凸显 |
| 7 | **上交所新增开户数**<br>(月度新增投资者) | **7%** | 辅助 | 月→日 | **散户入场情绪**。新增开户数是散户情绪的直接体现。牛市顶部往往伴随"开户潮"；底部则开户冷淡。数据截至 2023-08（akshare 上游停更），可通过 `account_open_user.csv` 手动补充。 | 开户数激增，"大妈入市"成为见顶信号 | 开户数极度低迷，市场无人问津 |
| 8 | **存款市值比**<br>(住户存款/A股总市值) | **7%** | 辅助 | 月→日 | **居民存款与股市市值的相对规模**。衡量"场外潜在购买力"。存款市值比高，说明居民有大量资金在场外等待入场；比值为历史低位时，说明资金已大量从存款搬家到股市。 | 存款市值比降至历史低位，居民杠杆加到极致 | 存款市值比较高，场外资金充裕但尚未入场 |
| 9 | **偏股基金三年年化收益率**<br>(中证偏股基金指数 930950) | **7%** | 辅助 | 日 | **专业机构投资者的长期回报率**。用滚动三年年化收益率衡量基金赚钱效应。收益率极高时往往对应市场顶部（均值回归），收益率极低甚至为负时往往是底部区域。 | 三年年化收益处于历史高位 → 均值回归压力大 | 三年年化收益为负或极低 → 长期布局窗口 |
| 10 | **股债基金比**<br>(股票型/债券型基金规模) | **7%** | 辅助 | 日 | **资金在股票基金和债券基金之间的配置偏好**。比值上升说明资金从债市流向股市，风险偏好提升；比值下降说明资金避险。反映了大类资产配置的迁徙方向。 | 股基规模远超债基，资金全面看多 | 债基规模占比上升，资金避险情绪浓厚 |

> **数据来源说明**：所有指标均通过 [akshare](https://github.com/akfamily/akshare) 免费开源库获取，无需任何付费API Token。历史数据频率为日度或月频（月频数据通过向前填充插值为日度）。

## 使用指南

### 1️⃣ 环境准备

```bash
# 克隆仓库
git clone <your-repo-url>
cd bull-market-heat-index

# 安装依赖
pip install -r requirements.txt
```

### 2️⃣ 初始化历史数据

```bash
# 采集全部10个指标的历史数据（首次运行，耗时较长）
python scripts/fetch_initial_hist.py

# 可选：只采集部分指标加速调试
python scripts/fetch_initial_hist.py --indicators turnover_rate,pe_valuation
```

### 3️⃣ 运行回测

```bash
python scripts/backtest.py
```

回测会：
- 计算 2005-01-01 至今的每日指数
- 生成走势图 `docs/chart.png`
- 验证5个关键历史时间点的阈值
- 输出回测报告

### 4️⃣ 每日更新（单日）

```bash
# 计算今日指数
python scripts/update_index.py

# 或指定日期
python scripts/update_index.py --date 2024-01-15
```

### 5️⃣ 邮件预警测试

```bash
# 配置 SMTP
export SMTP_HOST=smtp.qq.com
export SMTP_PORT=465
export SMTP_USER=your_email@qq.com
export SMTP_PASS=your_auth_code
export MAIL_TO=receiver@example.com

# 测试预警（强制发送）
python scripts/send_alert.py --force

# 预览邮件内容
python scripts/send_alert.py --dry-run
```

## 使用注意事项

### 权重的理解

采用 **双梯队加权**，核心指标（换手率、两融、PE、股债利差、成交额M2比、巴菲特指标）各 **12%**，辅助指标（开户数、存款市值比、基金年化、股债基金比）各 **7%**。

设计哲学是"给更快、更准的指标更高的敏感度"。如果某个指标在某段时期缺失数据，其权重会自动按比例分配给其他有效指标。例如：

- 若 **全部10个指标有数据** → 核心各12%，辅助各7%
- 若 **1个核心指标缺失**（如换手率） → 剩余 9 个指标的总权重 = 88%，每个有效指标实际权重 = 原始权重 / 88%

这意味着在早期年份（如 2008年），仅有成交额M2比和换手率2个核心指标有数据时，指数实际由这2个指标各贡献50%。随着时间推移，更多指标加入计算，指数维度逐渐丰富。

### 开户数的手动维护

上交所开户数据在 akshare 中停更于 **2023-08**。如需补充后续数据：

1. 打开 https://www.sse.com.cn/aboutus/publication/monthly/investor/
2. 找到"新增投资者数量"（万户），乘以 10000 转为"人"
3. 追加一行到 `data/raw/account_open_user.csv`：
   ```
   2024-01-01,1500000
   ```
4. 脚本会自动读取该文件并覆盖 akshare 数据

### 历史百分位的限制

- 百分位是**相对位置**，不是**绝对估值**。一个指标处于 90% 百分位，说明它比历史上 90% 的时间都高，但不意味着它"绝对过高"
- 百分位窗口是**全历史**（从该指标有数据的第一天起）。随着时间推移，新的极端值会改变历史百分位分布
- 部分指标仅从 2018 年起有数据（如沪深300 PE、偏股基金年化），因此这些指标的百分位仅反映近几年的相对位置

### 信号使用建议

| 指数范围 | 建议操作 |
|---------|---------|
| **≥80** 很热 | 🔴 减仓 / 对冲，不追高 |
| **60–80** 偏热 | 🟡 注意控制仓位，可逐步减仓 |
| **40–60** 适中 | ⚪ 正常仓位，趋势跟随 |
| **20–40** 偏冷 | 🟢 关注机会，可分批建仓 |
| **≤20** 很冷 | 🟣 历史极值区域，积极布局 |

> ⚠ 指数是**辅助参考工具**，不构成投资建议。任何单一指标都有滞后性，请结合基本面、政策面综合判断。

## GitHub Actions 自动化

配置好以下 GitHub Secrets 后，每天北京时间 20:00 自动运行：

| Secret | 说明 |
|--------|------|
| `SMTP_HOST` | SMTP 服务器地址 |
| `SMTP_PORT` | SMTP 端口（默认 465） |
| `SMTP_USER` | SMTP 用户名（邮箱） |
| `SMTP_PASS` | SMTP 授权码 |
| `MAIL_TO` | 预警接收邮箱 |
| `MAIL_FROM` | 发件邮箱（可选，默认=SMTP_USER） |
