## 4. 每日更新指数

### 4.1 更新今日指数

```bash
python scripts/update_index.py
```

流程说明：

1. 读取 `data/raw/` 下各指标的最新数据
2. 如果当日无新数据，自动沿用最近一期有效值
3. 对每个指标计算当前值在全历史中的百分位
4. 等权合成 → 得到今日指数
5. 追加到 `data/index_history.csv`
6. 生成简报图 `docs/chart.png`

### 4.2 指定日期更新（补历史）

```bash
# 补算某一天的数据
python scripts/update_index.py --date 2024-06-01
```

### 4.3 只生成简报不计算

如果仅需刷新图表（数据未变时）：

```bash
python scripts/briefing.py
```

### 4.4 更新日志解读


## 6. GitHub Actions 自动化部署

### 6.1 推送仓库

```bash
git add .
git commit -m "初始化牛市热度指数"
git remote add origin <你的GitHub仓库地址>
git push -u origin main
```

### 6.2 配置 Secrets

进入 GitHub 仓库 → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**：

| Secret | 说明 | 示例 |
|--------|------|------|
| `SMTP_HOST` | SMTP 服务器 | `smtp.qq.com` |
| `SMTP_PORT` | SMTP 端口 | `465` |
| `SMTP_USER` | 邮箱账号 | `your@qq.com` |
| `SMTP_PASS` | SMTP 授权码 | `xxxxxxxxxxxx` |
| `MAIL_TO` | 收件邮箱 | `alert@example.com` |
| `MAIL_FROM` | 发件邮箱（可选） | `your@qq.com` |

### 6.3 触发方式

| 方式 | 说明 |
|------|------|
| **定时触发** | 每个交易日北京时间 20:00（UTC 12:00）自动运行 |
| **手动触发** | GitHub → Actions → 每日牛市热度指数 → Run workflow |

### 6.4 工作流执行流程

```
检出仓库 → 安装依赖 → 采集数据 → 计算指数
  → 发送预警（如需） → 提交更新 → 推送
```

### 6.5 检查运行日志

GitHub → Actions → 点击最新工作流 → 展开各步骤查看详细输出

## 7. 自定义配置

### 7.1 修改权重

编辑 `scripts/config.py` 中的 `INDICATORS` 列表：

```python
INDICATORS: List[Indicator] = [
    Indicator(id="buffett_ratio", name="巴菲特指标", weight=0.15),  # 改为15%
    Indicator(id="turnover_m2", name="成交额M2比", weight=0.10),
    # ...
]
```

> ⚠ 所有权重之和应等于 1.0

### 7.2 修改热度等级阈值

编辑 `scripts/config.py` 中的 `HEAT_LEVELS`：

```python
HEAT_LEVELS = [
    (0, 25, "很冷", "..."),     # 原 0-20
    (25, 45, "偏冷", "..."),    # 原 20-40
    # ...
]
```

### 7.3 更换数据源

所有数据采集逻辑集中在 `scripts/fetch_initial_hist.py` 中，每个指标对应一个 `fetch_xxx()` 函数。如需更换数据源，修改对应函数即可。

### 7.4 邮件模板

邮件 HTML 模板在 `scripts/briefing.py` 的 `build_email_html()` 函数中，可自由修改样式和内容。

## 8. 常见问题与排错

### Q1: 数据采集时部分指标失败

**现象**：
```
[5/10] 大盘估值-PE (pe_valuation)
  获取沪深300估值数据…
  ✗ PE 采集失败: ...
```

**原因**：网络问题或 akshare 数据源暂时不可用。

**解决**：
```bash
# 稍后重试失败的指标
python scripts/fetch_initial_hist.py --indicators pe_valuation --overwrite

# 如果持续失败，检查网络
python3 -c "import akshare as ak; print(ak.stock_zh_index_hist_csindex('000300').head())"
```

### Q2: 图表中文显示为方框

**现象**：`docs/chart.png` 中的中文全部显示为 `□□□□`

**原因**：系统中缺少中文字体。

**解决**：

```bash
# macOS
brew install font-noto-sans-cjk

# Ubuntu (GitHub Actions 已自动安装)
sudo apt-get install fonts-noto-cjk-extra

# Windows
# 安装微软雅黑或宋体即可
```

### Q3: 指数突然变为极端值

**现象**：某天指数从 50 直接跳到 95 或跌到 5。

**检查**：
```bash
# 查看各指标当日百分位
tail -1 data/index_history.csv

# 查看是否有指标缺失
```

**常见原因**：某个关键指标当日数据异常或缺失，导致权重集中到其他指标。

### Q4: 邮件发送失败

**现象**：
```
❌ 邮件发送失败: (535, b'login fail')
```

**排查**：
1. 确认 SMTP 授权码正确（不是登录密码）
2. 确认 SMTP 端口匹配（QQ邮箱：465/SSL）
3. QQ邮箱需开启 SMTP 服务：设置 → 账户 → POP3/SMTP → 开启

### Q5: 如何补充缺失的历史数据

**背景**：部分指标（如 PE、股债利差）数据从 2018 年开始，早期年份无法覆盖。

**处理方式**：
- 系统自动将缺失指标的权重均分给有效指标
- 早期年份（如 2008）可能只有 2-3 个指标，指数反映的是这些有限维度的热度
- 随着时间推移，当更多指标加入计算，指数维度逐渐丰富
- 这是 v1.0 的设计约束，未来可补充更多历史数据源来改善

### Q6: GitHub Actions 运行失败

**检查步骤**：
1. 打开 GitHub → Actions → 点击失败的工作流
2. 查看具体报错步骤
3. 常见原因：
   - `pip install` 失败 → 检查 `requirements.txt`
   - 数据源不可达 → 重试即可
   - `git push` 失败 → 检查仓库权限

---

> 📖 **术语**
>
> - **百分位**：当前值在全部历史数据中的相对位置。90% 意味着比 90% 的历史数据都高。
> - **等权**：10个指标各占 10% 权重，无优先级之分。
> - **向前填充**：月频数据在当月所有日期使用同一个值，直到下个月新数据更新。
> - **滚动三年年化**：以过去约 750 个交易日为窗口计算的平均年化收益率。
