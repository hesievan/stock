

# 牛市热度指数需求说明书（正式版 v1.0）

## 1. 概述
**指数名称**：牛市热度指数（Bull Market Heat Index）  
**取值范围**：0 – 100  
**用途**：衡量A股市场整体冷热程度，辅助判断是否过热及逃顶时机。  
**更新频率**：每个A股交易日 **北京时间 20:00** 自动计算当日指数。  
**发布方式**：GitHub Actions 每日自动运行，结果存储于仓库 CSV 文件，并可触发邮件预警。

**热度等级表**：
| 指数区间 | 等级 | 解释 |
|---------|------|------|
| 0–20 | 很冷 | 历史极值低位，长期布局窗口 |
| 20–40 | 偏冷 | 市场偏冷，关注低估机会 |
| 40–60 | 适中 | 无显著超买超卖，常规仓位 |
| 60–80 | 偏热 | 短期过热风险上升，注意控仓 |
| 80–100 | 很热 | 逃顶风险显著，建议降低仓位 |

## 2. 指数计算方法
**公式**：  
`当日指数 = Σ (每个指标的当日历史百分位 × 权重)`  

- **权重**：10个指标等权，各 **10%**  
- **历史百分位定义**：`（小于当前值的历史数据个数）/（历史数据总个数）× 100`  
  - 严格小于当前值（等于时不计数），因此百分位取值范围 0–100，但理论上100仅在当前值大于所有历史值时出现。  
- **历史窗口**：从1990年12月（或该指标实际有数据的最早日期）至昨日。每日一个数据点。  
- **缺失值处理**：若某指标当日无法获取有效值（如数据源失败），则将其权重按比例均分给其余有效指标，保持当日总权重100%。

## 3. 指标清单及计算口径（10个，各10%权重）

| 序号 | 指标名称 | 权重 | 计算公式 / 数据口径 | 数据频率 | 缺失处理 |
|------|----------|------|---------------------|----------|----------|
| 1 | 巴菲特指标 | 10% | A股总市值 / 最近4个季度滚动GDP | 日（GDP季度） | 沿用最近一期 |
| 2 | 成交额M2比 | 10% | 两市日成交额 / 最新月末M2余额 | 日（M2月频） | 沿用最近一期 |
| 3 | 沪深换手率 | 10% | 全市场流通市值换手率 = 成交额 / 流通市值（akshare口径） | 日 | - |
| 4 | 两融余额市值比 | 10% | 两市融资融券余额 / 沪深总市值 | 日 | - |
| 5 | 大盘估值-PE | 10% | 沪深300 PE_TTM（中证指数规则：剔除负值，整体法） | 日 | - |
| 6 | 上交所个人开户数 | 10% | 月度新增开户数 | 月（每日沿用最近月值） | 沿用最近一期 |
| 7 | 股债利差 | 10% | (1 / 万得全A PE_TTM) - 10年期国债收益率 | 日 | - |
| 8 | 存款市值比 | 10% | 住户存款余额 / A股总市值 | 月（每日沿用最近月值） | 沿用最近一期 |
| 9 | 偏股型基金三年年化收益率 | 10% | **中证偏股基金指数（930950）** 的滚动三年年化收益率（每日基于过去三年日收益率计算） | 日 | 若不满三年历史则排除该日 |
| 10 | 股债基金比 | 10% | 股票型基金总规模 / 债券型基金总规模（证监会分类标准，混合型不纳入） | 月（每日沿用最近月值） | 沿用最近一期 |

**基金分类补充说明**：
- 股票型基金：按证监会标准，股票仓位≥80%的基金（含ETF、指数增强、LOF等）。
- 债券型基金：债券仓位≥80%的基金（含可转债基金）。
- 混合型基金：不纳入。
- 偏股基金指数（930950）直接替代第9个指标，无需自行计算仓位。

## 4. 数据源与免费实现方案

| 数据 | 推荐来源 | 说明 |
|------|----------|------|
| A股总市值、流通市值、成交额 | tushare (`market.TotalMV`, `market.CirculatedMV`, `index_daily`) | 需 token |
| 沪深300 PE_TTM、万得全A PE_TTM | tushare (`index_daily`) | 同上 |
| 融资融券余额 | tushare (`money.financial_indicator`) | 同上 |
| 10年期国债收益率 | tushare (`bond_benchmark_yield`) | 同上 |
| 中证偏股基金指数（930950）净值 | akshare (`fund_etf_hist_sina` 或 `fund_index_daily`) | 免费 |
| M2、住户存款余额 | akshare (`macro_china_money_supply`, `macro_china_depository_institutions_balance`) | 免费 |
| 上交所新增开户数 | akshare (`stock_account_statistic`) | 免费 |
| 基金规模数据（股债基金比） | akshare (`fund_scale_em`) 每日抓取天天基金网 | 免费，需反爬容错 |
| GDP | 国家统计局手动下载或 `akshare.macro_china_gdp_yearly` + 季度推算 | 免费，低频更新 |

> **注意**：基金规模数据每日通过 `akshare` 爬取天天基金网全量数据（耗时2-3分钟），若某日爬取失败则沿用前一日规模，并记录日志。

## 5. 历史百分位统一规则
- **各指标独立计算百分位**：从该指标有数据的首日开始。例如中证偏股基金指数（930950）最早约2004年，则百分位从2004年起算。
- **历史数据存储**：每日原始值存入 `data/raw/` 下对应的 CSV 文件。每日计算时读取全部历史，计算最新日的百分位并追加。
- **百分位不追溯重算**：历史百分位一旦计算存储，除非修正原始数据，否则不变。

## 6. 回测与验证
- **必须回测**：计算 2005年1月1日 至 昨日 的每日指数，生成曲线。
- **验证点**：
  - 2007.10 牛市顶部 → 预期指数 ≥80
  - 2008.12 底部 → 预期指数 ≤20
  - 2015.06 杠杆牛顶部 → 预期指数 ≥80
  - 2018.12 底部 → 预期指数 ≤20
  - 2021.02 茅指数顶部 → 预期指数 ≥70（偏热区间）
- 若偏差明显，可在后续版本调整权重或更换指标，但 v1.0 保持上述定义。

## 7. GitHub Action 自动化部署
- **触发时间**：每个交易日 **北京时间 20:00**（UTC 12:00）。  
  `cron: '0 12 * * 1-5'`  
- **运行环境**：ubuntu-latest, Python 3.10  
- **主要步骤**：
  1. 检出仓库（含历史数据文件）。
  2. 安装依赖（`requirements.txt` 含 `akshare`, `tushare`, `pandas`, `numpy`, `matplotlib`, `yagmail` 等）。
  3. 执行 `python scripts/update_index.py`（该脚本负责所有数据获取、百分位计算、指数合成、存储）。
  4. 若指数 ≥80 或 ≤20，调用 `scripts/send_alert.py` 发送邮件。
  5. 将更新后的 `data/index_history.csv` 及原始数据文件提交并推送回仓库。
- **密钥管理**：`TUSHARE_TOKEN` 存入 GitHub Secrets；邮件发送凭据也存入 Secrets。

## 8. 邮件预警规则
- **触发条件**：当日指数 ≥80 或 ≤20。
- **邮件内容**：包含当日指数值、热度等级、各指标百分位明细、简要评述。
- **邮件接收地址**：您需提供收件邮箱和 SMTP 服务器信息（如使用 QQ 邮箱或 Gmail）。
- **实现方式**：Python 标准库 `smtplib` + `email`，或使用 `yagmail` 简化。

## 9. 输出与存储
- **主文件**：`data/index_history.csv` 结构  
  `date,index_value,status,heat_desc,巴菲物,成交M2,换手率,两融市值,PE,开户数,股债利差,存款市值,基金三年年化,股债基金比`  
  其中各列存储对应指标的百分位（0–100）。
- **可视化（可选）**：每日生成一张热度仪表图，保存到 `docs/chart.png`，可随仓库推送。
